Intro to collaborative work · 2026

Your project has a home
here’s how to use it

🏢

One analogy to remember: think of GitHub as a shared office filing cabinet. Everyone on the team can access the same folders, take copies home to work on, and bring their changes back — without overwriting each other’s work. Three concepts are all you need to get started, and no technical background is required.

The three concepts

🗄️
Repository
The central hub

A repository (or “repo”) is a folder in the cloud that stores all your project’s files — code, documents, images — along with the entire history of every change ever made.

Anyone on the team can see the latest version of everything, who changed what, and when. You can always roll back to a previous version if something breaks.

📁 Repository
📄 index.html
🎨 styles.css
🖼️ logo.png
📝 README.md
🕰️ Full history
Plain terms: the repo is the single authoritative place where the project lives — the master binder everyone refers to.
⬇️
Pull
Getting the latest version

Pulling means downloading the most up-to-date version of the project from the repository onto your computer. You do this so you’re always working with the latest changes — including anything teammates have added.

Like walking to the office cabinet, grabbing a fresh copy of the document, and bringing it back to your desk.

🗄️ Repositorylatest version
PULL
💻 Your computeryour local copy
Plain terms: Pull = “give me the newest version.” Always pull before starting work to stay in sync with your team.
⬆️
Push
Sharing your changes

Pushing means uploading the changes you’ve made on your computer back to the repository, so everyone else on the team can see and use them.

Until you push, your changes only exist on your machine — nobody else can see them. Pushing is how you “save to the shared drive.”

💻 Your computerwith your changes
PUSH
🗄️ Repositoryupdated for all
Plain terms: Push = “here are my changes, share them with everyone.” It’s how your work becomes part of the official project.

Quick reference

Concept What it is When to use it Analogy
Repository The shared project folder in the cloud Always — it’s the source of truth The office master binder
Pull Download the latest version to your computer Before you start working each day Grabbing a fresh copy from the cabinet
Push Upload your changes back to the repository After you finish your changes Returning the updated document to the cabinet

The workflow in one line: Pull → make your changes → Push. Repeat every time you work on the project.
