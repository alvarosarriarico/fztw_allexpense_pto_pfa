Block 01

Git

Version control that powers GitHub and Claude Code

01

Git is the version control system that records every change you make to your code. Without it, none of the GitHub workflows or Claude Code commands in this guide will work. Installation takes about five minutes — and one setting during the process is critical for Claude Code to function correctly.

Download & run the installer

1. Open your browser and go to git-scm.com/download/win. The 64-bit Windows installer will download automatically.

2. Open the downloaded .exe file. Click through the first few screens — the defaults are correct.

3. When you reach the "Adjusting your PATH environment" screen, select the middle option:

☑  Git from the command line and also from 3rd-party software

This is the most important setting in the entire installer. It adds Git to your system PATH so that PowerShell, VS Code's built-in terminal, and Claude Code can all call Git commands without any extra configuration. If you choose "Use Git from Git Bash only" instead, Claude Code will not be able to find Git and commands like git commit will fail.

4. Continue clicking Next on all remaining screens — leave every other setting at its default. Click Install, then Finish.

Verify the installation

5. Close and reopen PowerShell (or any terminal) and run: git --version

You should see a response like git version 2.x.x.windows.1. This confirms Git is installed and available on your PATH.

💡  If the command returns "git is not recognized", close all terminal windows and reopen them — PATH changes only take effect in new sessions. If it still fails, re-run the installer and make sure the correct PATH option is selected.
02

Git global settings are stored in a single file on your machine and apply to every repository you work in. You only need to set these once. The commands below cover the settings that every developer needs from day one.

Identity

Git stamps your name and email on every commit you make. Open a terminal and run both commands, replacing the placeholder values with your own:

git config --global user.name "Your Full Name"

git config --global user.email "you@example.com"

💡  Use the same email address you registered with on GitHub. This is how GitHub links your commits to your profile and shows your contribution history correctly.

Default branch name

Set the default branch name to main so every new repository you create starts on the same branch name used by GitHub:

git config --global init.defaultBranch main

Default editor

Tell Git to open VS Code when it needs you to write or edit something (such as a commit message or a merge conflict). The --wait flag makes Git pause until you close the VS Code tab before continuing:

git config --global core.editor "code --wait"

Line ending handling

Windows and Mac/Linux use different line ending characters. This setting tells Git to automatically convert line endings to the Windows format when you check out files, and back to Unix format when you commit — preventing invisible differences from cluttering your diffs:

git config --global core.autocrlf true

Verify your settings

Run the following command to print every global setting that is currently configured:

git config --global --list

You should see your name, email, and the other values you just set listed in the output.

Block 02

VS Code

Your code editor and development environment

01

Visual Studio Code (VS Code) is a free code editor made by Microsoft. It is the industry standard for writing HTML, CSS, JavaScript, and almost any other language. Installation takes about three minutes and does not require any technical knowledge.

Installation steps

1. Open your browser and go to code.visualstudio.com.

2. Click the large blue Download for Windows button. A .exe file will download to your Downloads folder.

3. Open the downloaded file. If Windows shows a security prompt saying "Do you want to allow this app to make changes?", click Yes — this is expected and safe.

4. Click through the installer screens. When you reach the Select Additional Tasks screen, tick these two options before clicking Next:

  • ☑ Add "Open with Code" action to Windows Explorer context menu — lets you right-click any folder and open it directly in VS Code.
  • ☑ Add to PATH — lets you type code in PowerShell to open VS Code from the terminal.

5. Click Install, wait about 30 seconds, then click Finish. VS Code will open automatically.

💡  If VS Code does not open automatically, find it in your Start menu by searching Visual Studio Code. Pin it to your taskbar so it is always one click away.
02

VS Code opens to a Welcome tab the first time you launch it. A few quick settings will make your day-to-day experience much smoother — none of them require any technical knowledge to configure.

The interface at a glance

VS Code is divided into a few key areas you will use every day:

  • ☰ Activity Bar (left strip of icons) — switches between the file explorer, search, Git panel, and extensions.
  • 📄 Explorer (first icon, looks like two files) — shows all the files and folders in your project.
  • ✎ Editor (main area) — where you read and write code. You can have multiple files open as tabs.
  • ▶ Terminal (bottom panel) — a PowerShell window built into VS Code. This is where you run Git commands and Claude Code.

Recommended settings

Open your project folder. Go to File → Open Folder, navigate to your repository folder, and click Select Folder. VS Code remembers the last folder you opened and will return to it automatically next time.

Enable Auto Save. Go to File → Auto Save and click it to enable it. A tick will appear next to it. This saves your files automatically as you type so you never lose a change by forgetting to press Ctrl + S.

Open the terminal. Go to View → Terminal or press Ctrl + ` (the backtick key, just left of the 1 key). The terminal opens at the bottom of the screen, already pointing at your project folder — you do not need to cd anywhere when you open it this way.

Adjust font size. Hold Ctrl and scroll up or down inside the editor to make text larger or smaller instantly.

Use the Command Palette. Press Ctrl + Shift + P to open a search bar for every feature and setting in VS Code. If you ever cannot find something in the menus, type what you are looking for here.

💡  When you open the terminal inside VS Code, it automatically starts in your project folder — so you do not need to run cd first. This is the easiest way to avoid the "wrong folder" problem described in the Git & GitHub guide.
03

Connecting VS Code to your GitHub account lets you push and pull code, clone repositories, and review pull requests without ever leaving the editor. It also unlocks Settings Sync — your extensions, themes, and preferences follow you to any machine automatically.

Sign in to GitHub

1. Look at the very bottom-left corner of VS Code. You will see a person icon (the Accounts button). Click it.

2. Click Sign in with GitHub. A browser window will open asking you to authorise VS Code.

3. Click Authorize Visual-Studio-Code in the browser. GitHub will redirect you back to VS Code automatically. You may see a prompt asking "Allow this website to open VS Code?" — click Open.

4. Back in VS Code, the person icon now shows your GitHub username. You are signed in.

Enable Settings Sync

1. Click the person icon again and select Turn on Settings Sync…

2. A panel will appear listing what to sync — leave all options ticked and click Sign in & Turn on.

3. Choose GitHub when asked which account to use. Your settings are now backed up and will restore on any new machine the moment you sign in.

  • ☑ Extensions — every installed extension syncs automatically.
  • ☑ Keyboard shortcuts — your custom key bindings carry over.
  • ☑ Settings & theme — font size, colour theme, and all preferences.
  • ☑ Snippets — any custom code snippets you have saved.
💡  If you are on an Accenture-managed machine and the browser redirect does not return to VS Code automatically, copy the URL shown in the browser, paste it into VS Code's Command Palette (Ctrl + Shift + P), and run GitHub: Completing Authentication.
04

Extensions are add-ons that give VS Code new capabilities. Think of them like apps on your phone — VS Code is the phone, and extensions are the apps you install to do specific things. Each one installs in seconds and can be enabled or disabled at any time. The list below covers the extensions we recommend for this guide.

How to install an extension

Click the Extensions icon in the Activity Bar (the four squares on the left side), search for the extension name, and click Install. Or click the Install in VS Code link on any card below — it will open directly in the Extensions panel.

These extensions are not required to follow this guide or work with Git and GitHub. None of them affect your code's output. They are purely productivity tools — useful only if you spend a significant amount of time writing code inside VS Code and want a more polished editing experience. Feel free to skip them entirely or install them later once you are comfortable with the essentials.
